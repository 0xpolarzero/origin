# Origin Docs

This directory tracks the full development process for building a custom native app fork on top of opencode.

## Documentation Rules

- Every phase gets its own document in `docs/`.
- Each phase document must include:
  - goal and non-goals
  - exact scope boundaries
  - implementation steps
  - validation steps
  - acceptance criteria
  - risks and rollback notes
- If a decision changes, update the relevant phase doc and add a short "Decision Update" section with date and reason.
