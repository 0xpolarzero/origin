## Current Direction Baseline

- Repository remains pruned to desktop-relevant surfaces (Phase `01`).
- App startup defaults into a protected global workspace (`~/Documents/origin` by default) with global entry session flow (Phase `02`).
- Runtime/settings namespace is Origin-owned by default with no implicit OpenCode sharing; OpenCode portability is explicit import-only for providers/auth and model visibility toggles (Phase `03`).
- Runtime persistence/state transitions are machine-enforced with explicit transition legality and policy-lineage audit requirements (Phase `04`).
- JJ runtime operations are adapter-bound with deterministic bootstrap/read policy/run-workspace cleanup semantics and no-snapshot read guardrails (Phase `05`).
- User-facing branding should be `origin` for this app, with explicit upstream exceptions such as `OpenCode Zen` and provider IDs.
