## Current Direction Baseline

- Repository remains pruned to desktop-relevant surfaces (Phase `01`).
- App startup defaults into a protected global workspace (`~/Documents/origin` by default) with global entry session flow (Phase `02`).
- Runtime/settings namespace is Origin-owned by default with no implicit OpenCode sharing; OpenCode portability is explicit import-only for providers/auth and model visibility toggles (Phase `03`).
- Runtime persistence/state transitions are machine-enforced with explicit transition legality and policy-lineage audit requirements (Phase `04`).
- JJ runtime operations are adapter-bound with deterministic bootstrap/read policy/run-workspace cleanup semantics and no-snapshot read guardrails (Phase `05`).
- Workflow/library resources are YAML-authored with deterministic validation codes, standard-workspace capability blocking for unsupported resources/triggers, deterministic knowledge-base collision policy, and runtime run-gate rejection for non-runnable workflow definitions (Phase `06`).
- Manual workflow execution is orchestrated with per-run session linkage, deterministic duplicate-trigger rejection, bounded repair retries, run-workspace isolation/cleanup, and integration-candidate staging metadata persistence for change paths (Phase `07`).
- Integration is queue-orchestrated per workspace with immutable ready ordering, recovery/finalization of interrupted attempts, single stale-base replay, and deterministic cancel/timeout race settlement semantics (Phase `08`).
- History exposes deterministic Runs/Operations UX with cross-linking, provenance filtering, duplicate-event visibility, and stable pagination semantics (Phase `09`).
- Outbound drafts are adapter-owned envelopes with separate `dispatch_attempt` identity, send-time revalidation, origin-only outbound policy, strict managed-endpoint gating, and reserved deterministic validation surfaces `adapter_id=test`, `action_id=message.send`, and `action_id=issue.create` (Phase `10`).
- Approved future workflow direction is graph-first with AI-first authoring, immutable run snapshots, dedicated workflow/run/library surfaces, and sessions treated as linked artifacts rather than workflow containers ([15.graph-first-workflow-orchestration.spec.md](/Users/polarzero/code/projects/origin/docs/specs/15.graph-first-workflow-orchestration.spec.md)).
- User-facing branding should be `origin` for this app, with explicit upstream exceptions such as `OpenCode Zen` and provider IDs.

## Guardrails For Agents

- Read this file first for project intent and scope boundaries.
- Do not reintroduce removed non-desktop scope unless an approved phase spec explicitly requires it.
- Do not reintroduce implicit OpenCode fallback/sync or broad auto-migration behavior without an approved phase spec.
- Do not remove/bypass protected global workspace behavior without an approved phase spec.
- This product is pre-release; do not preserve legacy runtime data, backward-compatibility shims, or migration/backfill behavior unless a phase spec explicitly requires it.
- For future outbound work, use a generic Draft envelope plus adapter-owned actions/payload schemas and a centralized dispatcher; do not assume one universal payload shape or reuse JJ reconciliation attempts as the generic outbound attempt model.
- Keep changes aligned with native desktop goals and phase acceptance criteria.
