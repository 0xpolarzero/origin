# 02 Spec: Global Entry Sessions in a Protected Global Workspace

## Goal

Add a globally available command-palette entry (`Start agent session from entry`) that immediately creates and sends a new session in a dedicated global workspace, while making that workspace the default startup context and non-removable.

## Why This Phase Exists

The fork is moving toward multiple views layered on top of the current chat surface. Before adding new views, we need a reliable, global capture flow:

- type anywhere in the command palette
- create a fresh session
- send immediately
- do it in one predictable workspace (`~/Documents/origin` by default)

This keeps current session behavior intact while introducing a global entry point for future workflows.

## In Scope

- Decouple command-palette opening from `file.open` so the palette is available from app-level context, not only session context.
- Keep command registration context-aware (`register`/cleanup lifecycle), with global commands always available and session commands mounted only in session scope.
- Add a new command-palette synthetic entry item:
  - Title: `Start agent session from entry`
  - Visible only in full palette mode (`mode !== "files"`)
  - Visible only when query is non-empty and does not start with `/`
  - Ranked first for keyboard selection; `Enter` executes it by default
  - Includes a visual `origin` marker (badge/tag) aligned to the far right and a distinct icon.
- Entry action behavior:
  - Always create a fresh session
  - Always create it in the configured global workspace directory
  - Immediately send the typed text as the first prompt
  - Navigate to the newly created session.
- Add a new settings field for global workspace directory:
  - Single global directory value
  - Default: `~/Documents/origin` (resolved from `path.home`)
  - Configurable from Settings UI with directory picker/fallback selector.
- Ensure the global workspace directory exists:
  - Auto-create recursively when missing
  - Re-check/ensure before executing entry action.
- Startup behavior:
  - Treat app "no active folder" state as global workspace context
  - On startup, ensure/open global workspace and route into it.
- Protect the global workspace from removal:
  - Prevent close/remove actions for that workspace in UI and command paths.
- Validation and tests for ranking, routing, creation/send flow, and protection behavior.

## Out of Scope

- New top-level views/menus beyond this entry flow.
- Replacing slash command behavior in prompt input.
- Session content model redesign, response rendering changes, or chat UX overhaul.
- Multi-workspace policy engine beyond one protected global workspace.
- Full localization rollout for every new string in this phase (English-first keys acceptable with fallback behavior).

## Baseline Assumptions

- "Always uses the special session" means: always create a fresh new session inside the special global workspace (not a singleton reused session).
- The entry action should reuse currently selected model/agent; if unavailable, show existing "select model/agent" toast behavior and abort.
- Query beginning with `/` should preserve slash-command-oriented expectations; the new entry item must be suppressed in that case.
- Existing non-entry palette results and fuzzy matching should remain unchanged in ordering, except the new entry item gets top priority placement when shown.
- Global workspace default is derived from `path.home`; if `home` is unavailable, startup initialization is blocked with an error toast until path data is available.
- The global workspace may initially not exist; the app must create it automatically (`mkdir -p` equivalent semantics).

## Agent Execution Plan

## Step 1: Add Global Workspace Setting and Defaults

- Extend app settings schema with a single global workspace directory value.
- Default the value to `${path.home}/Documents/origin` when unset.
- Add Settings UI row in General for viewing/changing this directory.
- Add a "reset to default" interaction.
- Persist with migration-safe behavior (existing settings remain valid).

Suggested checks:

```bash
bun --cwd packages/app run test:unit -- src/context/settings.tsx
```

## Step 2: Add Directory Ensure API and Regenerate SDK

- Add backend endpoint for ensuring a directory exists (recursive create).
- Return normalized absolute path and explicit error payload on failure.
- Wire endpoint into JS SDK v2 generated client/types.
- Regenerate SDK with repo-required command:

```bash
bun ./packages/sdk/js/script/build.ts
```

## Step 3: Startup Global Workspace Bootstrap

- On app startup/root route entry:
  - Resolve configured global workspace path (or default).
  - Ensure directory exists via new API.
  - Open/register the workspace project and route into `/:dir/session` context.
- Keep Home as failure fallback only when bootstrap cannot complete.
- Ensure this bootstrap is idempotent and does not create route loops.

## Step 4: Protect Global Workspace From Removal

- Mark the configured global workspace as protected in app state derivation.
- Block close/remove flows for the protected workspace in:
  - UI actions (buttons/menu items hidden or disabled)
  - command/handler pathways (guard returns).
- Preserve all existing behavior for non-protected workspaces.

## Step 5: Decouple Palette Opening From Session `file.open`

- Introduce a globally available palette-open command path that does not depend on session command registration.
- Keep session command registration unchanged for session-only commands.
- Ensure command palette can open from app-level contexts after startup bootstrap.

## Step 6: Implement `Start agent session from entry`

- Add synthetic palette entry item in full mode only.
- Show when `query.trim().length > 0 && !query.trim().startsWith("/")`.
- Keep existing result computation unchanged; prepend this entry item so it is first-selected.
- Display:
  - title `Start agent session from entry`
  - `origin` badge/marker on the far right for reusable global entry affordance
  - distinct icon (magic/sparkle-like from current icon set).
- On select:
  - Ensure global workspace exists
  - Validate current agent/model availability
  - Create new session in global workspace
  - Send query text immediately via async prompt request
  - Navigate to created session route
  - Close palette.

## Step 7: Tests and Regression Coverage

- Add/extend unit tests for:
  - entry visibility and suppression (`/`-prefix, empty query, files-only mode)
  - entry-first ordering
  - global workspace protection guard
  - startup global workspace bootstrap decision logic.
- Add/extend e2e tests for:
  - app startup lands in global workspace context
  - entry action from palette creates new session and sends first message
  - action always targets global workspace even when invoked from another workspace
  - protected workspace cannot be removed.

## Risk Register

- Non-git workspace handling can surface path/worktree inconsistencies in project metadata-dependent UI.
- Startup bootstrap can create navigation loops if root redirect and invalid-dir recovery interact incorrectly.
- Directory auto-create can fail due permissions (readonly home/documents) and must degrade with explicit errors.
- Palette ranking changes can unintentionally affect muscle-memory command selection if first-item logic regresses.
- Global workspace protection guards may be bypassed by unguarded code paths unless all removal entry points are covered.
- Adding a new server endpoint requires SDK regeneration discipline; stale generated types can cause runtime mismatch.
